## 🎯 已实现的功能

### 后端 API

#### 认证相关
- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录
- `GET /api/auth/profile` - 获取个人信息（需要认证）

#### 活动相关
- `GET /api/events` - 获取活动列表（支持分页、搜索、筛选）
- `GET /api/events/:id` - 获取活动详情
- `POST /api/events` - 创建活动（需要认证）
- `PUT /api/events/:id` - 更新活动（需要管理员权限）
- `DELETE /api/events/:id` - 删除活动（需要管理员权限）

### 前端页面

- `/` - 首页（显示登录状态）
- `/login` - 登录页面
- `/register` - 注册页面
- `/events` - 活动列表页面
- `/events/[id]` - 活动详情页面

### UI组件

- `Button` - 按钮组件
- `Input` - 输入框组件
- `Card` - 卡片组件
- `EventCard` - 活动卡片组件

## 🛠 技术实现

### 后端技术栈
- Node.js + Express + TypeScript
- Prisma ORM
- PostgreSQL 数据库
- JWT 认证
- bcrypt 密码加密

### 前端技术栈
- Next.js 15 (App Router)
- React 18
- TypeScript
- Tailwind CSS
- Context API (状态管理)

### 数据库
- 4个核心表：User, Event, DJ, Genre
- 完整的索引和约束
- 支持全文搜索

## 🚀 如何运行

### 1. 启动数据库
```bash
docker-compose up -d
```

### 2. 启动后端（终端1）
```bash
cd server
pnpm dev
# 运行在 http://localhost:3001
```

### 3. 启动前端（终端2）
```bash
cd web
pnpm dev
# 运行在 http://localhost:3002
```

### 4. 创建测试数据
```bash
./test-auth.sh      # 创建测试用户
./seed-events.sh    # 创建测试活动
```
