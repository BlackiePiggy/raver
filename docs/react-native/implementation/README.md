# Raver React Native Implementation Plans

> Purpose: 存放每个执行单元的局部落地方案。每个页面、feature 或底座能力开始实现前，先在这里写清楚 iOS 来源、RN 策略、checkbox 和验收标准。

## Phase Directories

```text
phase-01-foundation/
phase-02-auth-shell/
phase-03-discover-events-music/
phase-04-community-feed/
phase-05-profile-search-checkins/
phase-06-notifications/
phase-07-messages-squads/
phase-08-release-parity/
```

## Rules

- 每个执行单元一份文档。
- 文档必须引用 iOS source。
- 文档必须列出是否需要用户确认。
- 实现前 checklist 应处于 `Draft` 或 `Active`。
- 收口后状态改为 `Done`，未进入当前阶段的能力改为 `Deferred`。
