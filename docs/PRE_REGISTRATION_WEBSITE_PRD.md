# 预登记（预注册）网站需求文档（V2）
> 更新时间：2026-04-21  
> 产品：RaveHub

## 1. 已确认需求输入
- 产品名称：`RaveHub`
- 首页标题：`RaveHub电子音乐共建社区`
- 首页副标题：`欢迎Raver们回家～`
- 中部模块：需要展示产品功能 + 配套 UI 图像
- 手机号：支持国际号码，国家区号前缀 `+XX` 下拉选择
- 内测资格规则：由后台人工决定（不是自动随机）
- 通知方式：多端都做（邮件/短信/微信/站内）
- 目标上线时间：`2026-05-31`（五月底）

## 2. 项目目标
- 建立一个围绕“预登记 + 内测资格抽取”的落地页。
- 强化“抽取内测资格”稀缺叙事，提升登记转化与用户分享意愿。
- 配套管理后台，支持人工抽取、批次管理、全渠道通知、数据分析。

## 3. 核心业务策略（稀缺性）
- 用户提交预登记后，不代表立即获得内测资格。
- 网站需在首屏、登记区、成功页、FAQ 明确写出：`内测资格将按批次抽取发放。`
- 提交成功态必须包含抽取提示（必选文案）：
  - `预登记完成，我们会尽快告知内测资格。`
  - `预登记完成，请等待抽取内测资格，请静候佳音。`

## 4. 前台用户流程
1. 用户进入首页，看到价值主张和 `去登记` 按钮。
2. 点击 `去登记`，页面平滑滚动到最下方登记区（`#pre-register`）。
3. 用户向下浏览中段功能模块（多屏滚动，图文结合）。
4. 用户在底部提交预登记表单。
5. 提交成功后，登记区替换为“预登记完成 + 等待抽取内测资格”状态。

## 5.4 底部预登记区（核心表单）
- 锚点：`#pre-register`
- 字段：
  - 邮箱（必填）
  - 手机号（可选）：`国家区号下拉(+XX)` + 号码输入框
  - 微信号（可选）
  - 称呼（必选）：`Miss.`、`Mr.`、`先生`、`女士`
  - 对产品的期望留言（可选）
- 按钮文案：`提交预登记`
- 合规提示：`提交即表示你同意我们联系你并告知内测资格结果。`

## 5.5 提交成功态
- 表单区切换为成功卡片：
  - 主文案：`预登记完成，请等待抽取内测资格，我们会尽快通知你。`
  - 次文案：`内测名额有限，感谢你的耐心等待。`

## 6. 表单规则与防刷
- 邮箱：必填、格式校验、转小写、去空格。
- 手机号：选填，采用 `country_code + national_number` 结构存储。
- 国际号码校验：优先按 E.164 规则进行基础校验。
- 微信号：选填，字符范围 `字母/数字/_/-`。
- 留言：上限 500 字。
- 防重复：邮箱唯一；重复提交提示“该邮箱已完成预登记”。
- 防刷：IP 限流 + 行为风控，验证码作为可配置开关。

## 7. 管理后台（必须）

## 7.1 登录与权限
- 角色建议：
  - `Admin`：全部权限
  - `Operator`：查看、筛选、导出、通知

## 7.2 预登记管理
- 列表字段：邮箱、手机号、微信号、称呼、留言摘要、来源、状态、提交时间。
- 筛选：状态、时间区间、渠道来源、联系方式完整度。
- 搜索：邮箱/手机号/微信号。
- 导出：CSV。

## 7.3 内测资格抽取（人工决策）
- 后台创建“抽取批次”（批次名、计划名额、备注）。
- 从候选池中由运营人工勾选中签名单。
- 支持批量设置：
  - `SELECTED`（已获得资格）
  - `NOT_SELECTED`（本轮未中签）
  - `WAITLIST`（候补）
- 审计留痕：操作人、操作时间、变更前后状态、备注理由。

## 7.4 多端通知中心
- 目标：对中签/未中签/候补用户进行多端触达。
- 渠道：
  - 邮件（必做）
  - 短信（必做）
  - 微信（必做，具体通道待接入策略确认）
  - 站内通知（Web/App 消息中心）
- 能力：
  - 模板管理（中签/未中签/候补）
  - 批次发送
  - 发送结果回执与重试队列

## 7.5 运营看板
- 核心指标：
  - 累计预登记人数
  - 每日新增
  - 联系方式完整度
  - 各渠道中签率
  - 通知送达率/失败率

## 8. 数据模型（建议）

## 8.1 `pre_registrations`
- `id` uuid
- `email` unique not null
- `phone_country_code` nullable（示例：+86、+1）
- `phone_number` nullable
- `wechat_id` nullable
- `salutation` enum(MISS, MR, XIAN_SHENG, NV_SHI)
- `expectation_message` text nullable
- `status` enum(SUBMITTED, SELECTED, NOT_SELECTED, WAITLIST, NOTIFIED)
- `source` nullable
- `created_at`
- `updated_at`

## 8.2 `beta_draw_batches`
- `id` uuid
- `batch_name`
- `planned_slots`
- `note`
- `created_by`
- `created_at`

## 8.3 `beta_draw_decisions`
- `id` uuid
- `batch_id`
- `registration_id`
- `decision` enum(SELECTED, NOT_SELECTED, WAITLIST)
- `decision_by`
- `decision_reason` nullable
- `created_at`

## 8.4 `beta_notifications`
- `id` uuid
- `registration_id`
- `batch_id`
- `channel` enum(EMAIL, SMS, WECHAT, IN_APP)
- `template_key`
- `send_status` enum(PENDING, SENT, FAILED)
- `provider_message_id` nullable
- `error_message` nullable
- `created_at`

## 9. API（建议）
- `POST /api/pre-registrations`：提交预登记
- `GET /api/admin/pre-registrations`：后台分页查询
- `POST /api/admin/beta-batches`：创建抽取批次
- `POST /api/admin/beta-batches/:id/decisions`：人工设置中签/未中签/候补
- `GET /api/admin/beta-batches/:id/results`：查看批次结果
- `POST /api/admin/notifications/send`：多渠道批量通知
- `GET /api/admin/notifications/logs`：通知发送日志
