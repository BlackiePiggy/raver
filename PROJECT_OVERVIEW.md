# Raver - 电子音乐爱好者平台

## 项目概述
面向电子音乐爱好者的综合社交平台，提供活动信息、打卡集邮、音乐分类、歌单分享等功能。

## 技术栈
- **前端**: React 18 + Next.js 14 + TypeScript + Tailwind CSS
- **移动端**: React Native + TypeScript (iOS优先)
- **后端**: Node.js + Express + TypeScript
- **数据库**: PostgreSQL + Prisma ORM
- **认证**: JWT + OAuth 2.0
- **存储**: AWS S3 / 阿里云OSS
- **部署**: Vercel (前端) + Railway/Render (后端)

## 设计风格
- **主题**: 暗黑模式/科技风
- **色彩方案**:
  - 主色: 深紫色 (#8B5CF6) / 电蓝色 (#3B82F6)
  - 背景: 深灰黑 (#0F0F0F, #1A1A1A)
  - 强调色: 荧光绿 (#10B981) / 荧光粉 (#EC4899)
  - 文字: 白色/浅灰 (#FFFFFF, #E5E5E5)
- **设计原则**: 简洁、高对比度、流畅动画、卡片式布局

## 核心功能模块

### 1. 电音资讯 (Events)
- 国际/国内音乐节信息聚合
- 活动日历视图
- 地图定位功能
- 活动筛选(地区、时间、风格、艺人)
- 活动提醒推送

### 2. 打卡集邮 (Check-in)
- **活动打卡**: 记录参加过的音乐节/活动
  - 时间、地点、场次信息
  - 手动标注听的Set类型
  - 照片上传
  - 个人活动时间线
- **DJ集邮**: 官方DJ库打卡
  - DJ个人页面
  - 观看记录
  - 收藏功能
- **成就系统**: 徽章、等级、统计数据

### 3. 电子音乐分类 (Genres)
- 风格树状图展示
- 每个风格的详细介绍
- 典型曲目推荐
- 代表DJ列表
- 风格演进历史

### 4. 新歌速递 (New Releases)
- 每周四新歌整合
- 按风格分类
- 试听功能(接入Spotify/Apple Music API)
- 收藏/分享功能

### 5. 未发ID讨论区 (Unreleased IDs)
- 帖子发布(音频/视频上传)
- 评论讨论
- 投票/点赞
- 标签系统
- 搜索功能

### 6. Set歌单 (Setlists)
- DJ表演信息录入
- 时间戳对应歌单
- 视频链接关联
- 用户贡献审核
- 歌单导出(Spotify/Apple Music)

### 7. 社交功能 (Social)
- 用户注册/登录
- 个人主页
- 头像/昵称设置
- **粉丝牌系统**:
  - DJ粉丝牌(可挂多个)
  - 风格粉丝牌(可挂多个)
  - 牌子等级显示
- 关注/粉丝系统
- 动态发布
- 私信功能

## 数据库设计要点
- Users (用户表)
- Events (活动表)
- DJs (DJ表)
- Genres (风格表)
- CheckIns (打卡记录)
- Setlists (歌单表)
- Tracks (曲目表)
- Posts (帖子表)
- Badges (徽章表)
- UserBadges (用户徽章关联)

## 开发阶段
1. **Phase 1**: 项目初始化、设计系统、数据库设计
2. **Phase 2**: 核心功能开发(活动、打卡、DJ库)
3. **Phase 3**: 社交功能、用户系统
4. **Phase 4**: 高级功能(歌单、未发ID、新歌速递)
5. **Phase 5**: iOS App开发
6. **Phase 6**: 测试、优化、上线
